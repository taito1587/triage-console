# 06. 入賞戦略（加点設計）

> 本ドキュメントは「どう勝つか」の単一情報源。Zenn 記事の骨子にもそのまま使う。

## 大会の評価軸（確定事実）

- **テーマ：「業務の"課題"をAIエージェントでいかに"鮮やかに"解決するか」→ 企業の業務改革**
- 必須：**Azure 実行基盤 or Copilot Studio + Microsoft AI 技術**（→ Foundry 構成で条件クリア）
- 提出物：**成果物 URL + Zenn 記事 + GitHub(任意)** ← Zenn 記事が審査の主資料
- 個人部門：最優秀 50万 / 優秀 30万 / 特別 20万 / 奨励 10万×2
- クレジット：個人 **$200 相当**（自主予算 $98 はこの内枠）
- ⚠️ **提出締切は要確認**（ページ解析では 6/1 23:59 と出た。事実なら計画前提が変わるため公式で要確認）

## 現プランの弱点 → 対策

テーマは「**解決**」と「**業務改革**」を問うている。素のMVPは "助言止まり" かつ ROI の物語がないため、ここを埋める。

| 弱点 | 対策（採用済み加点要素） |
|------|------------------------|
| 助言で終わり、"解決"していない | **A. アクション実行**（保全へTeamsエスカレーションをAgentが自律実行） |
| デモの鮮やかさが弱い | **B. 画像解析**（GPT-4o vision で故障部位写真を解析） |
| 業務改革・ROI の物語がない | **C. ROIストーリー**（ダウンタイム削減を定量化、画面5+記事） |

## 採用した加点要素（2026-06-01 決定）

### A. アクション実行（Teams 通知 / チケット起票）★必須級
- 緊急度 High（または品質影響あり）と判定したとき、Agent が **保全チームへ Teams 通知**を出す。
- 実装：**Teams Incoming Webhook** をバックエンドから叩く（$0）。発展形は Foundry の action tool（Logic Apps / Azure Functions）。
- メッセージ内容：設備 / 症状 / 緊急度 / 原因候補Top3 / 推奨初動 / トリアージ結果へのリンク。
- 価値：「助言」→「実行」。Agent が業務を**動かす**ことを示し、テーマの「解決」に直接応える。

### B. マルチモーダル入力（GPT-4o vision）★強く推奨
- 異常入力フォームに**写真添付**（既存の任意項目を活用）。
- GPT-4o vision で画像を解析し、Intake の構造化に反映（例：摩耗痕・異物・エラー表示パネルの読取り）。
- 価値：デモの山場。現場リアル。低コスト（トークンのみ）。

### C. 業務改革・ROI ストーリー ★必須級（主に記述）
- **問題の定量化**：「ライン異常の初動判断に平均 N 分／熟練者依存／夜間は属人化」など現場課題を数値で提示。
- **効果の定量化**：本 Agent で初動を M 分短縮 → ダウンタイム削減 → 年間 ◯◯円相当。
- 画面5（ナレッジ集計）に **削減ダウンタイム / 平均初動時間 KPI** を表示。記事の冒頭と結論にも置く。
- 数値はデモ用の想定値で良いが、出典・前提を明記して「それっぽさ」でなく「筋の通った試算」にする。

### D/F/E/G（記事・小工数で補強）
- **D. 自律的なツール選択**：オーケストレータは固定パイプにせず、緊急度に応じ escalation tool を自分で呼ぶ／呼ばないを判断する。「スクリプトでなく Agent」を担保。
- **F. 賢くなる実測**：フィードバック前後で確信度・的中が上がる様子を可視化（簡易メトリクス）。
- **E. Foundry Evaluation**：groundedness / relevance を計測し記事に掲載（差別化ネタ）。
- **G. Responsible AI**：出典必須・幻覚抑制・Content Safety を記事で明記。

## 優先方針

**コアMVP（5画面）+ A + B を並行で進める。** C は記述中心なので並行で仕込む。
締切が逼迫した場合の死守ラインは「動くコア + A + B + ROI記述」。D/F/E/G は余裕分。

## 減点・地雷の回避

- 「キレイな RAG 検索」に見せない → A（実行）・D（自律分岐）で回避。
- **デプロイ未完／動かないが最大の減点** → Day 1 で provision を前倒し、早期に疎通を確保。
- **Zenn 記事の質が直接スコア** → 図（mermaid 済）・デモ GIF・ROI・技術的工夫・学びを厚く書く。
